# Geterdone Configuration Schema

This document defines the structure of `geterdone-config.json`, the central configuration file created and maintained by the Setup skill. All Geterdone skills reference this schema when reading user preferences, installed connectors, and scheduled workflows.

## File Location

```
project-root/
├── geterdone-config.json       (auto-generated by setup)
└── ...
```

## Field Definitions

### Root Level

**version** (string, required)
- Configuration schema version
- Current: `"1.0"`
- Used by: All skills to ensure compatibility
- Updates: Increment when breaking schema changes occur

**configuredAt** (ISO 8601 timestamp, required)
- Initial setup timestamp
- Used by: Setup skill to determine first-run behavior
- Format: `"YYYY-MM-DDTHH:MM:SSZ"`

**lastUpdated** (ISO 8601 timestamp, required)
- Most recent modification timestamp
- Used by: All skills to validate configuration freshness
- Updated by: Any skill that modifies the config

### user Object

Stores user profile and workflow preferences.

**workflowType** (string, required)
- Primary type of work the user does
- Values: `CODE`, `CONTENT`, `OUTREACH`, `ANALYTICS`, `TEAM`, `ALL`
- Used by: Connector recommendation engine to suggest relevant tools
- Set by: Setup skill during discovery phase

**primaryRole** (string, optional)
- User's job title or primary function
- Examples: "Engineer", "Content Creator", "Sales Manager"
- Used by: Skills that customize messages or workflows for role-specific contexts
- Set by: Setup skill user profile step

**timezone** (string, required)
- User's primary timezone for scheduling
- Format: IANA timezone identifier (e.g., `"America/New_York"`)
- Used by: Schedule management, notification timing
- Set by: Setup skill

**displayName** (string, optional)
- User's preferred name for personalization
- Used by: All skills for greetings and notifications
- Set by: Setup skill

### dispatch Object

Configures task execution and automation behavior.

**preferredRunner** (string, required)
- Automation platform preference for scheduled tasks
- Values: `cowork`, `n8n`, `zapier`, `make`, `other`
- Default for Defusely: `cowork` (uses Cowork's `create_scheduled_task` tool)
- Used by: Auto-Execute skill to route workflows
- Set by: Setup skill based on installed connectors

**enableAutoExecute** (boolean, required)
- Whether auto-execute tasks run on schedule
- Default: `false` (user must explicitly enable)
- Used by: Auto-Execute skill gating
- Set by: Setup skill, configurable by user

**autoExecuteTime** (string, HH:MM format)
- Default time for auto-execute tasks if not specified per-task
- Example: `"09:00"`
- Used by: Auto-Execute skill as fallback schedule
- Set by: Setup skill

**taskQueue** (string, optional)
- Reference to task queue system if using one
- Example: `"celery"`, `"rq"`, `"sqs"`
- Used by: Advanced scheduling skills
- Set by: User configuration

### execution Object

Controls how tasks and workflows execute.

**timeoutSeconds** (number, required)
- Maximum seconds before task execution times out
- Default: `300` (5 minutes)
- Used by: All execution skills to prevent hangs
- Set by: Setup skill, user can override per-task

**retryAttempts** (number, required)
- Number of retries if task fails
- Default: `3`
- Used by: Auto-Execute and dispatch skills
- Set by: Setup skill based on workflow criticality

**errorNotification** (string, required)
- Where to send task failure notifications
- Values: `slack`, `email`, `none`
- Used by: All execution skills
- Set by: Setup skill based on installed connectors

**verboseLogging** (boolean, optional)
- Whether to log detailed execution traces
- Default: `false`
- Used by: Debug and monitoring skills
- Set by: User preference

### memory Object

Configures knowledge base and context management.

**primaryProvider** (string, required)
- Main tool for storing notes and knowledge
- Values: `notion`, `obsidian`, `evernote`, `other`
- Used by: Context retrieval, knowledge integration
- Set by: Setup skill based on installed connectors

**contextWindowSize** (number, required)
- Maximum tokens to include in context for LLM operations
- Default: `8000`
- Used by: Context assembly skills
- Set by: Setup skill, adjustable by user

**autoSaveInterval** (number, required)
- Seconds between automatic context saves
- Default: `60`
- Used by: Auto-save background jobs
- Set by: Setup skill

### integrations Object

Tracks all installed and available connectors.

**connectors** (array, required)
- List of connector configurations
- Each connector has:
  - `name` (string): Official connector name
  - `uuid` (string): MCP registry UUID
  - `enabled` (boolean): Whether this connector is active
  - `installed` (boolean): Whether connector is installed
  - `category` (string): Connector type
  - `credentials` (string or null): Reference to secrets storage
  - `settings` (object): Connector-specific configuration

**installed_count** (number, required)
- Count of enabled connectors
- Used by: UI display, statistics
- Auto-calculated by: Setup skill

**available_count** (number, required)
- Total connectors available in MCP registry
- Used by: Upgrade recommendations
- Auto-calculated by: Setup skill via registry search

### schedules Object

Manages scheduled tasks and automation.

**tasks** (array, required)
- List of scheduled task configurations
- Each task has:
  - `id` (string): Unique task identifier
  - `name` (string): Human-readable task name
  - `type` (string): `morning_brief`, `auto_execute`, or `custom`
  - `enabled` (boolean): Whether task runs
  - `frequency` (string): `daily`, `weekly`, `monthly`, or `custom`
  - `time` (string): HH:MM format
  - `timezone` (string): Task-specific timezone override
  - `connectors` (array of UUIDs): Which connectors this task uses
  - `config` (object): Task-specific settings

### templates Object

Manages workflow and response templates.

**enabled** (boolean, required)
- Whether template system is active
- Default: `true`
- Used by: Skills that generate responses

**defaultTemplate** (string, optional)
- Name of default response template
- Used by: Skills when no specific template specified
- Set by: User preference

**customTemplates** (array, optional)
- References to user-defined templates
- Used by: Custom workflow skills
- Set by: User via template manager

## Schema Usage Across Skills

### Setup Skill
- **Creates**: Entire config on first run
- **Updates**: All fields when re-run
- **Preserves**: Existing connector data, schedules, custom templates

### Morning Brief Skill
- **Reads**: `schedules.tasks` (morning_brief type)
- **Reads**: `integrations.connectors` (enabled ones)
- **Reads**: `user.timezone`
- **Updates**: `lastUpdated` timestamp

### Auto-Execute Skill
- **Reads**: `dispatch.enableAutoExecute`, `dispatch.preferredRunner`
- **Reads**: `execution.timeoutSeconds`, `execution.retryAttempts`
- **Reads**: `schedules.tasks` (auto_execute type)
- **Updates**: `lastUpdated` timestamp

### Context Skills
- **Reads**: `memory.primaryProvider`, `memory.contextWindowSize`
- **Reads**: `integrations.connectors[].settings`
- **Reads**: `user.workflowType` (for context relevance)

### Connector Installation
- **Reads**: `integrations.connectors` (for current state)
- **Updates**: `integrations.connectors[]` (add/enable/disable)
- **Updates**: `integrations.installed_count`
- **Updates**: `lastUpdated`

## Validation Rules

1. **Required Fields**: All root-level required fields must exist
2. **UUID Format**: Connector UUIDs must match MCP registry format
3. **Timestamp Format**: All timestamps must be ISO 8601 with Z timezone
4. **Enums**: All enum fields must match specified values
5. **Timezone**: Must be valid IANA timezone identifier
6. **Time Format**: All HH:MM times must be valid (00:00 - 23:59)

## Migration & Versioning

When schema version increments:
1. Setup skill detects version mismatch
2. Runs migration logic to update config structure
3. Preserves all existing user data
4. Updates version field
5. Sets lastUpdated timestamp

This ensures Setup can be re-run safely without losing user configuration.
